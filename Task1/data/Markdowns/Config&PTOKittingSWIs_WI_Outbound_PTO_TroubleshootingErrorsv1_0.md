## **PTO – TROUBLESHOOTING ERRORS**

Below is a list of potential errors and resolutions for issues you may encounter as part of the PTO Kitting and Overpack activities using the Packing Workbench.

| Error message                                       | Resolution                                                                                                                                                                                                                              |
|-----------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Duplicate Kit/Overpack Labels                       | This may happen in the following cases:                                                                                                                                                                                                 |
|                                                     | -<br>When someone else is updating the Delivery from another computer. In that case both<br>users should log out of Oracle and access it again                                                                                          |
|                                                     | -<br>When both PTO Operators on the same workstation simultaneously scan the 'Validate &<br>Close' option for a Kit and Overpack.                                                                                                       |
|                                                     | <br>Always verify the serial number printed on the Kit Label with the actual item before<br>applying the label to the Kit box                                                                                                          |
| Corrupted LPN                                       | This issue can be seen in the following ways:                                                                                                                                                                                           |
|                                                     | 1.<br>Occasionally, when scanning an LPN, the system will not accept this LPN. When this<br>happens, use a new LPN instead.                                                                                                             |
|                                                     | 2.<br>It also occurs that once you have started using the LPN and have already linked one or more<br>kit LPNs ('NLK%' or 'TXK%') to this LPN, the LPN will fail to perform the merge-up.                                                |
|                                                     | •<br>If that is the case do NOT perform a breakdown as this will cause more data<br>corruption.                                                                                                                                         |
|                                                     | •<br>Instead use the mobile application (scangun) to merge the problem LPN into a new<br>LPN                                                                                                                                            |
|                                                     | •<br>If that also fails, you will need to unpack the kit LPNs ('NLK%' or 'TXK%') from the<br>faulty LPN one by one, again using the mobile application (scangun)                                                                        |
| '%1Pxxxxx' error when scanning the item             | You may see this error when the item also exists as a serial number.                                                                                                                                                                    |
| number<br>('xxxxx' represents the item number)      | The configuration for the scangun (sheet with all the barcodes) includes the workaround for this. So, to<br>resolve this error, please make sure the scangun is configured correctly.                                                   |
|                                                     | Ideally, the inventory records should be updated to prevent this error. To resolve this please contact the<br>Inventory Control Team so that they can correct the Serial Number.                                                        |
|                                                     | The 'Zebra Bad Serial Number Scan Details Report 'can be used (by the Inventory Control Team) to pro<br>actively check for incorrect serial numbers                                                                                     |
|                                                     |                                                                                                                                                                                                                                         |
| Merge-up (Reconfigure) process takes a<br>long time | Please be patient: For large PTO Orders this process may take longer e.g., 10 minutes.<br>If it takes too long, then please contact the AMS Support Team, so that they can check if there are any<br>blocking locks or locked sessions. |
| Overpack Qty shows 0                                | This is a bug in the system, that has not yet been resolved. The labels may print '0' as to Overpack<br>Quantity for some of the labels. Often the quantity is printed correctly again after a few more<br>overpacks have been created. |

LPN error when scanning the item number The item also exists as an LPN number So, the Inventory Control Team will need to correct the LPN Number The 'Zebra LPN Name Update Data Fix' program can be used to rename the incorrect LPN number. This program works only for those LPNs which have status "Issued out of stores" or "Defined but not used" The program - after running - adds the prefix 'LPN00001' to the LPN number, e.g. LPN00001-nl85137950 No eligible material found This error will be displayed when there is no inventory in the Packing Station Location in subinventory SYS1.
